# Design Notes

このリポジトリでは、30分のハンズオンで見せる小さな管理画面を作る。

画面は派手にしない。SaaSの管理画面として、何が起きているかすぐ読めることを優先する。

## UIの方向

- 1画面で完結する小さな管理画面にする
- 最初に状態が分かるヘッダーを置く
- 操作は右側、状態や名前は左側に寄せる
- 数字は比較しやすいように桁をそろえる
- 色は状態を伝えるために使い、装飾だけのグラデーションは使わない
- カードは必要な単位だけに使う。ページ全体をカードで囲まない

## コンポーネント

- shadcn/ui の生成物は `components/ui` に置く
- Base UI はアクセシブルな挙動が欲しい部品に使う
- アイコンが必要なら lucide-react を使う
- 既存の `Button` と `cn` を優先する

## ハンズオン中の判断

完成度より、Codex Appの流れが伝わることを優先する。

参加者に見せたいのは、きれいな最終画面そのものではなく、方針を渡して、実装して、ブラウザで見て、差分をレビューする流れ。
